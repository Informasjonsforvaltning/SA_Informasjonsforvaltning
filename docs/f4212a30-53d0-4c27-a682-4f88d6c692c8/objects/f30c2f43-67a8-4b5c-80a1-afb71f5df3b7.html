Scenario 1 - Frittstående åpne data API med betaling 1. En eier av et åpent datasett ønsker å tilby et API til omverdenen. 2. Eieren etablerer et API for datasettet som tilbys gjennom en API-tilbyder 3. API-tilbyder oppretter en API-beskrivelse i sin API-manager (lokal API-katalog) eller direkte som en Open API Specification fil 4. API-et høstes av API-katalog, og gjøres tilgjeneglig som en søk og oppslagstjeneste 5. API-konsument finner API-et i API-katalogen, og basert på OAS beskrivelsen setter igang utvikling av klient-applikasjon 6. Klient-applikasjon, nå satt i produksjon, kaller API-et.
