Scenario 1.1 - Frittstående åpne data API - En eier av et åpent datasett ønsker å tilby et API til omverdenen. 1. Datasetteieren etablerer et API for datasettet som tilbys fra en API-tilbyder 2. API-tilbyder oppretter en API-beskrivelse direkte som en Open API Specification fil 3. API-beskrivelsen høstes maskinelt av felles API-katalog, og gjøres tilgjeneglig som en søk og oppslagstjeneste 4. Tjenesteier (applikasjons-/tjenesteutvikler) finner API-et i API-katalogen. 5. Basert på OAS beskrivelsen setter igang utvikling av konsument-applikasjon og kall til API-et 6. Sluttbruker benytter en tjeneste og konsument-applikasjon, som nå satt i produksjon, kaller API-et.
